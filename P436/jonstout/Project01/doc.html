Summary

Philosopher Problem

The dining philosophers problem is summarized as five philosophers sitting at a table doing one of two things: eating or thinking. While eating, they are not thinking, and while thinking, they are not eating. The five philosophers sit at a circular table with a large bowl of rice in the center. Chopsticks are placed in between each pair of adjacent philosophers, and as such, each philosopher has one Chopstick to his left and one fork to his right. As Rice is difficult to serve and eat with a Chopstick, it is assumed that a philosopher must eat with two Chopsticks. Each philosopher can only use the Chopsticks on his immediate left and immediate right.

Worker Problem

In this problem, there is a pool of work to be done, and we have a pool of threads to use to do it. The pool of work should be drawn from using a rst in rst out policy (it is a queue). Furthermore, the workers require exclusive access to shared resources to complete the tasks - the resources that are needed are task dependent. Each task takes a certian amount of time, at which point the worker can proceed to a new task. New work enters the queue at random intervals. If the queue is empty, workers should wait brie y, then check for more work.

How to Build

  • Philosopher: make philosopher
  • Dead Philosopher: make deadPhilosopher
  • Worker Problem: make workerProblem

Software Usage

  • Philosopher:
      1. To run philosopher type "./philosopher"
      2. Args: <seconds to run>, <number of philosophers>(optional)
  • Dead Philosopher:
      1. To run deadPhilosopher type "./deadPhilosopher"
      2. Args: <seconds to run>, <number of philosophers>(optional)
  • Worker Problem:
      1. To run philosopher type "./philosopher"
      2. Args: <number of workers>(optional), <initial amount of work>(optional) <time till new work>(optional), <number of resources>(optional) <max number of resources per job>(optional), <maximum amount of time a task should take>(optional) <maximum size of work queue>(optional)

Know Bugs/Issues

    Issues

  • Variable Print Outs at the beginning of each philosopher program is hard to see. In order to see them you must exit the program before more lines are printed than are allowed by your shell.

Question Answers

  • If the philosophers are in deadlock, they will continue to be in the "THINKING" state forever.
  • If one or more philosophers are starving, we will see some stuck in the "THINKING" state while others remain in the "EATING" state.
  • If the program is run for an extended period deadlock always occoured in my dead lock implementation of the philsopher problem.
  • The philosophers all ate for the same amount of time for my dead lock free implementation of the philosopher problem.
